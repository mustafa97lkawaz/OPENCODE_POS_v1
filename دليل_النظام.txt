أولاً: التغييرات التي تمت على النظام
(Changes Made to the System)
================================================================

1. ميزة الطباعة المباشرة (Direct USB Thermal Printing)
   -------------------------------------------------------
   - تم إضافة دعم الطباعة المباشرة على الطابعة الحرارية (Thermal Printer)
     عبر USB بدون أي نافذة حوار (Print Dialog)
   - يتم الطباعة تلقائياً بعد إتمام كل عملية بيع
   - يمكن إعادة طباعة أي فاتورة من صفحة المبيعات (Sales History)
   - تم إضافة زر "طباعة تجريبية" في صفحة الإعدادات (Settings)
   - تم إضافة ملف (electron/rawprint.ps1) لإرسال بيانات ESC/POS
     مباشرة عبر Windows Print API بوضع RAW

2. تحديث شاشة البيع (POS Screen)
   -------------------------------------------------------
   - تم تحويل إرسال الفاتورة من (Form Submit) إلى (AJAX)
   - بعد إتمام البيع يتم الطباعة تلقائياً بدون إعادة تحميل الصفحة
   - في حال فشل الطباعة تظهر رسالة تحذير بدون إيقاف عملية البيع

3. تحديث صفحة الإعدادات (Settings Page)
   -------------------------------------------------------
   - تم إضافة حقل "اسم الطابعة" (Printer Name)
   - زر "تحديث القائمة" يجلب قائمة الطابعات المثبتة على Windows
   - زر "طباعة تجريبية" للتحقق من الاتصال بالطابعة

4. إصلاح تطبيق Electron
   -------------------------------------------------------
   - تم إصلاح خطأ ELECTRON_RUN_AS_NODE في ملف package.json
   - تم نقل معالجات ipcMain داخل app.whenReady()
   - تم إضافة ملف CLAUDE.md لتوثيق المشروع

5. تنظيف مستودع Git (Repository Cleanup)
   -------------------------------------------------------
   - تم حذف الملفات الكبيرة (dist/, electron-app/node_modules/)
     من تاريخ Git كاملاً
   - تم تحديث .gitignore لاستبعاد الملفات التنفيذية والمجلدات الكبيرة

================================================================
ثالثاً: الإعداد الأول (First Time Setup)
================================================================

الخطوة 1: تشغيل XAMPP
  - افتح XAMPP Control Panel
  - شغّل Apache (Start)
  - شغّل MySQL (Start)

الخطوة 2: إعداد قاعدة البيانات
  - افتح المتصفح وادخل: http://localhost/phpmyadmin
  - أنشئ قاعدة بيانات جديدة باسم: MustafaPos
  - استورد الملف (إن وجد): database/database.sql

الخطوة 3: إعداد ملف البيئة (.env)
  - افتح المجلد: C:\xampp\htdocs\pos_opencodee\
  - انسخ الملف .env.example وسمّه .env
  - عدّل القيم التالية:

      APP_NAME=mustafaInvoices
      APP_URL=http://localhost
      DB_DATABASE=MustafaPos
      DB_USERNAME=root
      DB_PASSWORD=

الخطوة 4: تثبيت المكتبات (Install Dependencies)
  - افتح Command Prompt في مجلد المشروع
  - نفّذ الأوامر التالية:

      composer install
      php artisan key:generate
      php artisan migrate
      npm install

الخطوة 5: بناء ملفات CSS و JavaScript
      npm run production


================================================================
رابعاً: تشغيل التطبيق (Running the Application)
================================================================

--- الطريقة الأولى: تشغيل Electron (التطبيق المكتبي) ---

المسار: C:\xampp\htdocs\pos_opencodee\

1. تأكد أن XAMPP يعمل (Apache + MySQL)

2. شغّل Electron عبر الأمر التالي في Command Prompt:

      npm run electron:start

   أو لتشغيل XAMPP و Electron تلقائياً معاً:

      npm run launch

ملاحظة: لا تشغّل Electron قبل تشغيل Apache في XAMPP
         وإلا ستظهر شاشة بيضاء أو خطأ في الاتصال

--- الطريقة الثانية: تشغيل عبر المتصفح (Web Browser) ---

1. تأكد أن XAMPP يعمل (Apache + MySQL)
2. افتح المتصفح وادخل:

      http://localhost/pos_opencodee/public

--- الطريقة الثالثة: خادم التطوير (Development Server) ---

      php artisan serve

   ثم افتح: http://127.0.0.1:8000

خامساً: إعداد الطابعة الحرارية (Thermal Printer Setup)
================================================================

--- نوع الطابعة المدعومة ---
  - أي طابعة حرارية (Thermal Printer) تعمل بـ USB
  - مثال: XP-80 / Epson TM-T88 / Star TSP100 / Bixolon
  - حجم الورق المدعوم: 80mm أو 58mm

--- الخطوة 1: تثبيت الطابعة على Windows ---
  1. صل الطابعة بالحاسوب عبر كابل USB
  2. افتح: Settings (إعدادات Windows)
     → Bluetooth & devices (البلوتوث والأجهزة)
     → Printers & scanners (الطابعات والماسحات)
  3. انقر "Add a printer or scanner"
  4. انتظر حتى يظهر اسم الطابعة ثم انقر "Add device"
  5. بعد التثبيت تأكد من ظهور اسم الطابعة في القائمة
     مثال: XP-80

--- الخطوة 2: معرفة اسم الطابعة الدقيق ---
  1. افتح: Settings → Printers & scanners
  2. انقر على الطابعة → Properties
  3. احفظ الاسم كما يظهر بالضبط
     مثال: XP-80 أو EPSON TM-T88V

--- الخطوة 3: إعداد الطابعة في النظام ---
  1. افتح التطبيق (Electron أو المتصفح)
  2. من القائمة الجانبية اختر: الإعدادات (Settings)
  3. في حقل "اسم الطابعة (USB / Windows)" اكتب:
     اسم الطابعة كما يظهر في Windows بالضبط
     مثال: XP-80
  4. اختر نوع الورق:
     - 80mm  ← للطابعات العريضة (الأكثر شيوعاً)
     - 58mm  ← للطابعات الصغيرة
  5. اضغط "حفظ الإعدادات"

--- الخطوة 4: اختبار الطابعة ---
  1. في صفحة الإعدادات اضغط "تحديث القائمة"
     سيظهر اسم طابعتك في قائمة منسدلة
  2. اضغط "طباعة تجريبية"
  3. إذا طبعت الطابعة = الإعداد صحيح ✓
  4. إذا ظهرت رسالة خطأ = تحقق من اسم الطابعة

--- حل المشاكل الشائعة (Troubleshooting) ---

  المشكلة: "فشل إرسال البيانات للطابعة"
  الحل:
  - تأكد أن اسم الطابعة مكتوب بشكل صحيح في الإعدادات
  - افتح: Settings → Printers & scanners وتحقق من الاسم
  - تأكد أن الطابعة متصلة وتعمل (لا توجد أخطاء في قائمة Windows)

  المشكلة: الطابعة تطبع رموز غريبة بدل النص
  الحل:
  - تأكد من اختيار نوع الورق الصحيح (80mm أو 58mm)
  - تأكد أن الطابعة تدعم ESC/POS

  المشكلة: لا تطبع بعد إتمام البيع
  الحل:
  - تأكد أنك حفظت إعدادات الطابعة
  - جرب "طباعة تجريبية" أولاً للتحقق من الاتصال

--- إعادة طباعة فاتورة سابقة (Reprint) ---
  1. من القائمة الجانبية اختر: المبيعات (Sales)
  2. ابحث عن الفاتورة المطلوبة
  3. اضغط زر الطابعة (أيقونة خضراء) بجانب الفاتورة
  4. ستطبع الفاتورة مباشرة

سابعاً: أوامر مفيدة للمطورين (Developer Commands)
================================================================

تشغيل التطبيق:
  npm run electron:start      ← تشغيل Electron
  npm run launch              ← تشغيل XAMPP + Electron
  php artisan serve           ← خادم التطوير على port 8000

بناء الأصول (Assets):
  npm run dev                 ← بناء للتطوير
  npm run watch               ← مراقبة التغييرات تلقائياً
  npm run production          ← بناء للإنتاج (مضغوط)

قاعدة البيانات:
  php artisan migrate         ← تشغيل migrations
  php artisan migrate:fresh   ← حذف وإعادة إنشاء الجداول
  php artisan db:seed         ← إدخال بيانات تجريبية
  php artisan cache:clear     ← مسح الـ Cache
  php artisan config:clear    ← مسح إعدادات الـ Cache
  php artisan route:list      ← عرض جميع الـ Routes

مسارات الطباعة (Print Routes):
  GET /print/receipt/{id}     ← طباعة فاتورة
  GET /print/test             ← طباعة تجريبية
  GET /print/printers         ← قائمة الطابعات
  GET /print/debug/{name}     ← تشخيص مشاكل الطابعة
