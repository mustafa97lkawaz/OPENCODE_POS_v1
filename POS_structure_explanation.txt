--------------------------------------------------------------------------------
1. نظرة عامة على هيكل نظام POS
--------------------------------------------------------------------------------

نظام نقطة البيع (POS) في هذا المشروع مبني باستخدام Laravel ويحتوي على:

- View (الواجهة): resources/views/sales/pos.blade.php
- Controller (التحكم): app/Http/Controllers/SaleController.php
- Routes (المسارات): routes/web.php

--------------------------------------------------------------------------------
2. Routes (المسارات)
--------------------------------------------------------------------------------

المسارات المتعلقة بـ POS موجودة في ملف routes/web.php:

# Route الرئيسية لصفحة POS
GET /pos - عرض شاشة نقطة البيع
Route: Route::get('pos', [SaleController::class, 'pos'])->name('pos');

# Route الوضع ملء الشاشة
GET /pos/fullscreen - عرض شاشة POS ملء الشاشة
Route: Route::get('pos/fullscreen', [SaleController::class, 'posFullscreen'])->name('pos.fullscreen');

# Route حفظ البيع
POST /sales - حفظ عملية البيع
Route: Route::resource('sales', SaleController::class);

# Route تعليق البيع
POST /sales/suspend - تعليق عملية البيع
Route: Route::post('sales/suspend', [SaleController::class, 'suspend'])->name('sales.suspend');

# Route عرض المبيعات المعلقة
GET /suspended-sales - عرض قائمة المبيعات المعلقة
Route: Route::get('suspended-sales', [SuspendedSaleController::class, 'index'])->name('suspended.index');

# AJAX Routes - جلب المنتجات

# جلب المنتجات حسب الفئة
GET /pos/products - جلب جميع المنتجات أو حسب الفئة
Route: Route::get('pos/products', [SaleController::class, 'getProducts'])->name('pos.products');

# البحث عن المنتجات
GET /pos/products/search - البحث بالمنتج بالاسم أو الباركود
Route: Route::get('pos/products/search', [SaleController::class, 'searchProducts'])->name('pos.products.search');

# جلب منتج بالباركود
GET /pos/products/barcode/{barcode} - جلب منتج معين بالباركود
Route: Route::get('pos/products/barcode/{barcode}', [SaleController::class, 'getProductByBarcode'])->name('pos.products.barcode');

--------------------------------------------------------------------------------
3. SaleController (التحكم)
--------------------------------------------------------------------------------

الملف: app/Http/Controllers/SaleController.php

الطرق (Methods):

1. pos() - عرض شاشة POS
   - تجلب جميع المنتجات النشطة مع فئاتها
   - تجلب جميع الفئات
   - تجلب العملاء النشطين
   - ترجع View: sales.pos

2. posFullscreen() - وضع ملء الشاشة
   - نفس وظيفة pos() لكن تعرض ملء الشاشة

3. store(Request $request) - حفظ عملية البيع
   - التحقق من صحة البيانات
   - إنشاء رقم فاتورة تلقائي: INV-YYYYMMDDHHMMSS
   - حساب المجموع والضريبة والخصم
   - إنشاء سجل في جدول Sales
   - إنشاء سجلات الأصناف في جدول SaleItems
   - خصم الكمية من المخزون

4. destroy(Request $request) - حذف/إلغاء عملية بيع
   - إعادة الكميات للمخزون
   - حذف السجل

5. suspend(Request $request) - تعليق عملية بيع
   - حفظ المنتجات في جدول suspended_sales
   - إنشاء رقم معرف: SUSP-YYYYMMDDHHMMSS

6. getProducts(Request $request) - AJAX جلب المنتجات
   - تقبل category_id كمعامل
   - ترجع JSON بالمنتجات

7. searchProducts(Request $request) - AJAX البحث
   - تقبل q (الاستعلام)
   - البحث في: اسم المنتج، الباركود، SKU

8. getProductByBarcode($barcode) - AJAX جلب بالباركود
   - البحث في جدول المنتجات
   - إذا لم يُfound يبحث في جدول المتغيرات (variations)

4. هيكل صفحة POS (View)
--------------------------------------------------------------------------------

الملف: resources/views/sales/pos.blade.php

# القسم الأول: الترويسة (Header)
- حقل البحث عن منتج (يدعم الماسح الضوئي)
- اختيار العميل (Select2)

# القسم الثاني: لوحة المنتجات (Products Panel)
- tabs الفئات: عرض جميع الفئات
- grid المنتجات: عرض المنتجات بشكل بطاقات
- كل بطاقة تحتوي على:
  * صورة المنتج
  * اسم المنتج
  * سعر البيع

# القسم الثالث: لوحة السلة (Cart Panel)
- عنوان: "السلة"
- قائمة الأصناف المضافة
- ازرار التحكم بالكمية (+ / -)
- ازرار حذف المنتج
- الإجماليات:
  * المجموع الفرعي
  * الضريبة (15%)
  * الخصم
  * الإجمالي الكلي

# القسم الرابع: ازرار التحكم
- زر دفع (F2)
- زر مسح السلة (Esc)
- اختصارات لوحة المفاتيح

# Modal الدفع (Payment Modal)
- اختيار طريقة الدفع:
  * نقدي (Cash)
  * بطاقة (Card)
  * تقسيم (Split)
- حقل المبلغ المستلم
- عرض الباقي
- زر تأكيد الدفع

--------------------------------------------------------------------------------
5. آلية إضافة المنتجات للسلة
--------------------------------------------------------------------------------

كيفية عمل إضافة المنتجات:

# الطريقة الأولى: النقر على بطاقة المنتج
1. المستخدم ينقر على بطاقة المنتج
2. تستدعي الدالة: addToCartDirect(productId, name, price, stock)
3. تتحقق من توفر المخزون
4. إذا كان المنتج موجوداً تزيد الكمية
5. إذا لم يكن موجوداً تضيفه للسلة
6. تستدعي renderCart() لإعادة رسم السلة

# الطريقة الثانية: البحث وقراءة الباركود
1. المستخدم يكتب في حقل البحث أو يمرر الماسح الضوئي
2. عند الضغط على Enter يتم استدعاء: handleBarcodeScan(barcode)
3. ترسل طلب AJAX لمسار: /pos/products/barcode/{barcode}
4. تتحقق من وجود المنتج
5. تستدعي addToCart() لإضافته للسلة

# الطريقة الثالثة: البحث النصي
1. المستخدم يكتب اسم المنتج
2. بعد توقف الكتابة (300ms) ترسل طلب: /pos/products/search?q=...
3. تعرض النتائج في الشبكة
4. المستخدم ينقر على المنتج لإضافته

--------------------------------------------------------------------------------
6. آلية حساب الأسعار والضرائب
--------------------------------------------------------------------------------

المتغير: TAX_RATE = 0.15 (15%)

# دالة updateTotals()
1. تحسب المجموع الفرعي: مجموع (سعر × كمية) لكل منتج
2. تحسب الضريبة: المجموع الفرعي × 0.15
3. تقرأ قيمة الخصم من الحقل
4. تحسب الإجمالي: المجموع + الضريبة - الخصم
5. تعرض القيم في الصفحة

--------------------------------------------------------------------------------
7. آلية عملية الدفع
--------------------------------------------------------------------------------

1. المستخدم ينقر زر "دفع" أو F2
2. تفتح نافذة الدفع (Modal)
3. يختار طريقة الدفع:
   - نقدي: يُدخل المبلغ ويحسب الباقي
   - بطاقة: المبلغ الكامل
   - تقسيم: يُدخل جزء نقدي وبطاقة

4. عند تأكيد الدفع:
   - تجمع كل البيانات في نموذج مخفي (Hidden Form)
   - ترسل طلب POST لـ /sales
   - Controller يحفظ في قاعدة البيانات
   - يعرض رسالة نجاح

--------------------------------------------------------------------------------
8. تعليق واستئناف المبيعات
--------------------------------------------------------------------------------

# تعليق البيع (Suspend):
1. المستخدم ينقر "تعليق" أو F4
2. تحفظ السلة في جدول suspended_sales
3. يُفتح رقم معرف Sale لاستئنافه لاحقاً

# استئناف البيع:
1. المستخدم يذهب لـ /suspended-sales
2. ينقر "استئناف" على عملية معينة
3. تُعاد المنتجات للسلة
4. يكمل العملية

--------------------------------------------------------------------------------
9. نماذج البيانات (Data Models)
--------------------------------------------------------------------------------

# جدول Sales (المبيعات)
- id
- invoice_number (رقم الفاتورة)
- customer_id (العميل)
- subtotal (المجموع)
- tax_amount (الضريبة)
- discount (الخصم)
- total (الإجمالي)
- payment_method (طريقة الدفع)
- cash_amount (المبلغ نقدي)
- card_amount (المبلغ بطاقة)
- paid_amount (المبلغ المدفوع)
- change_due (الباقي)
- Status (الحالة)
- Created_by (منشئ الفاتورة)

# جدول SaleItems (بنود الفاتورة)
- id
- sale_id (معرف البيع)
- product_id (معرف المنتج)
- qty (الكمية)
- unit_price (سعر الوحدة)
- total (الإجمالي)

# جدول suspended_sales (المبيعات المعلقة)
- id
- invoice_number
- customer_id
- items_json (منتجات السلة كـ JSON)
- total
- Created_by

--------------------------------------------------------------------------------
10. ملخص تدفق البيانات
--------------------------------------------------------------------------------

1. المستخدم يدخل للصفحة /pos
2. Controller يجلب المنتجات والفئات والعملاء
3. الصفحة تعرض المنتجات كـ Grid
4. المستخدم يضيف منتج للسلة (نقر/بحث/باركود)
5. JavaScript يحتفظ بالمنتجات في Array
6. يتم حساب الإجماليات في جانب العميل
7. المستخدم يختار العميل والدفع
8. يرسل البيانات للـ Controller
9. Controller يحفظ في قاعدة البيانات
10. يعرض رسالة نجاح
